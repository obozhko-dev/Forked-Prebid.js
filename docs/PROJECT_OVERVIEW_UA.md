## 🎯 Огляд Проекту

Комплексне рішення header bidding побудоване на базі фреймворку Prebid.js з кастомним bid адаптером для програматичної реклами. Цей проект розширює open-source бібліотеку Prebid.js спеціалізованим bid адаптером для рекламної мережі **Obozhko**, забезпечуючи інтеграцію real-time bidding (RTB) та серверну доставку креативів для оптимальної монетизації реклами.

## 📋 Архітектура Схем та Валідація

### **Валідація Bid Запитів**
Проект реалізує комплексну валідацію bid через специфічні схеми адаптерів:

```javascript
// Валідація Obozhko Адаптера
isBidRequestValid: function(bid) {
  return !!bid.params && 
         !!bid.params.anonId && 
         !!bid.sizes;
}
```

### **Патерни Схем Запит/Відповідь**

#### **Схема Запиту Obozhko**
```javascript
{
  anonId: string,           // Обов'язково: Анонімний ID користувача
  adUnitCode: string,       // Обов'язково: Ідентифікатор рекламного слоту  
  sizes: Array<[number, number]>, // Обов'язково: Розміри реклами
  pageUrl: string,          // Опціонально: URL сторінки для контексту
  userAgent: string,        // Опціонально: Ідентифікація браузера
  geo: string,             // Опціонально: Географічний таргетинг
  bidId: string            // Обов'язково: Унікальний ідентифікатор bid
}
```

#### **Валідація Схеми Відповіді**
```javascript
// Валідація Відповіді Obozhko
{
  requestId: bid.bidId,     // Має співпадати з bidId запиту
  cpm: number,             // Вартість за тисячу показів
  width: number,           // Ширина креативу
  height: number,          // Висота креативу
  creativeId: string,      // Унікальний ідентифікатор креативу
  currency: string,        // Код валюти (USD, EUR, тощо)
  ad: string,             // HTML контент реклами
  ttl: number,            // Час життя в секундах
  meta: object            // Додаткові метадані
}
```

### **Переваги та Використання Схем**
- **Безпека Типів:** Інтеграція TypeScript забезпечує валідацію під час компіляції
- **Валідація Виконання:** Prebid.js валідує всі bid відповіді проти очікуваних схем
- **Запобігання Помилкам:** Невалідні bids автоматично фільтруються перед аукціоном
- **Консистентний Інтерфейс:** Стандартизована комунікація між адаптерами та основним движком
- **Підтримка Відлагодження:** Чіткі повідомлення про помилки для невалідних форматів bid

## ⚡ Основні Технології та Бібліотеки

### **Фреймворк Header Bidding**
- **Prebid.js** - Індустріальний стандарт фреймворку header bidding
  - **Чому:** Open-source, vendor-neutral, підтримує 300+ demand партнерів глобально
  - **Переваги:** Real-time bidding, управління timeout, інтеграція аналітики, розширена екосистема адаптерів
  - **Особливості:** Паралельна обробка аукціонів, контроль детальності цін, управління згодою користувача, визначення пристрою

### **Система Збірки та Управління Модулями**
- **Gulp** - JavaScript task runner для автоматизації збірки
  - **Чому:** Зріла система збірки з розширеною екосистемою плагінів для складних робочих потоків збірки
  - **Переваги:** Stream-базована обробка, паралельне виконання завдань, спостереження за файлами, генерація source map
  - **Особливості:** Компіляція ES6 модулів, мініфікація коду, автоматизація тестів, сервер розробки

- **Webpack** - Модульний bundler та оптимізація активів
  - **Чому:** Передові можливості дозволу модулів, розділення коду та оптимізації
  - **Переваги:** Tree shaking, оптимізація chunks, hot module replacement, управління активами
  - **Особливості:** ES6 import/export, динамічні імпорти, архітектура плагінів, інструменти розробки

- **Babel** - JavaScript транспайлер для крос-браузерної сумісності
  - **Чому:** Дозволяє використовувати сучасні JavaScript особливості зберігаючи браузерну сумісність
  - **Переваги:** Підтримка ES6+ синтаксису, автоматична ін'єкція polyfill, трансформації на базі плагінів
  - **Особливості:** Конфігурації preset, оптимізація runtime, збереження source map, підтримка TypeScript

### **Тестування та Забезпечення Якості**
- **Mocha** - Багатофункціональний фреймворк тестування JavaScript
  - **Чому:** Зрілий фреймворк тестування з відмінною підтримкою async та гнучким звітуванням
  - **Переваги:** Підтримка BDD/TDD, паралельне виконання тестів, кастомні репортери, браузерне тестування
  - **Особливості:** Синтаксис describe/it, хуки (before/after), контроль timeout, фільтрація тестів

- **Chai** - Бібліотека асерцій з експресивним синтаксисом
  - **Чому:** Читабельні асерції які роблять тести самодокументованими та підтримуваними
  - **Переваги:** Множинні стилі асерцій (expect, should, assert), екосистема плагінів, повідомлення про помилки
  - **Особливості:** Глибока перевірка рівності, асерції типів, валідація властивостей, підтримка промісів

- **Sinon** - Standalone test spies, stubs та mocks
  - **Чому:** Комплексні утиліти тестування для ізоляції залежностей та тестування поведінки
  - **Переваги:** Stubbing функцій, відстеження викликів, fake timers, XMLHttpRequest mocking
  - **Особливості:** Створення spy, контроль поведінки stub, очікування mock, ізоляція sandbox

- **Karma** - Test runner для тестування в реальних браузерах
  - **Чому:** Тести запускаються в справжніх браузерах забезпечуючи реальну сумісність
  - **Переваги:** Мульти-браузерне тестування, підтримка continuous integration, звіти покриття коду
  - **Особливості:** Плагіни запуску браузерів, підтримка препроцесорів, live reload, віддалене тестування

### **Якість Коду та Стандарти**
- **ESLint** - Конфігурований JavaScript linter
  - **Чому:** Забезпечує стандарти якості коду та виявляє потенційні помилки на ранній стадії
  - **Переваги:** Настроювані правила, auto-fixing, інтеграція IDE, командна консистентність
  - **Особливості:** Створення кастомних правил, екосистема плагінів, наслідування конфігурацій, підтримка TypeScript

- **TypeScript** - Статична перевірка типів для JavaScript
  - **Чому:** Додає безпеку типів під час компіляції до JavaScript зменшуючи runtime помилки
  - **Переваги:** Краща підтримка IDE, безпека рефакторингу, самодокументований код, поступове впровадження
  - **Особливості:** Визначення інтерфейсів, generic типи, декларації модулів, строга перевірка null

### **HTTP Комунікація та Мережа**
- **XMLHttpRequest** - Браузерний HTTP клієнт для bid запитів
  - **Чому:** Прямий доступ до браузерного API з повним контролем обробки запит/відповідь
  - **Переваги:** Відсутність зовнішніх залежностей, promise-based async патерни, контроль timeout
  - **Особливості:** Кастомні заголовки, JSON серіалізація, обробка помилок, парсинг відповідей

### **Утилітарні Бібліотеки**
- **dlv** - Безпечний глибокий доступ до властивостей вкладених об'єктів
  - **Чому:** Запобігає runtime помилкам при доступі до вкладених властивостей об'єктів
  - **Переваги:** Null-safe навігація, підтримка значень за замовчуванням, легка реалізація
  - **Особливості:** Шляхи dot notation, доступ до індексів масивів, fallback значення, збереження типів

- **dset** - Швидке присвоєння глибоких властивостей об'єктів
  - **Чому:** Ефективна маніпуляція вкладених об'єктів для конфігурації та обробки даних
  - **Переваги:** Присвоєння на основі шляхів, автоматичне створення об'єктів, обробка масивів
  - **Особливості:** Шляхи dot notation, присвоєння масивів, збереження типів, контроль мутацій

- **klona** - Швидка утиліта глибокого клонування
  - **Чому:** Безпечне дублювання об'єктів для обробки bid без спільного використання посилань
  - **Переваги:** Обробка циклічних посилань, кастомні стратегії клонування, оптимізація продуктивності
  - **Особливості:** Глибоке копіювання, обробка масивів, збереження дат, клонування функцій

### **Криптографія та Безпека**
- **crypto-js** - JavaScript криптографічна бібліотека
  - **Чому:** Клієнтське шифрування та хешування для безпечної обробки даних
  - **Переваги:** Множинні алгоритми, безпечна генерація випадкових чисел, підтримка HMAC
  - **Особливості:** AES шифрування, SHA хешування, Base64 кодування, виведення ключів

  - **Особливості:** Об'єкти запит/відповідь, значення перерахувань, схеми валідації, визначення типів

- **iab-adcom** - Реалізація стандарту IAB AdCOM
  - **Чому:** Спільна об'єктна модель для обміну рекламними даними

  - **Особливості:** Контекстні об'єкти, медіа специфікації, регуляторна відповідність, таксономія даних

- **iab-native** - Специфікація IAB Native Advertising

  - **Особливості:** Визначення активів, формати відповідей, керівництва рендерингу, специфікації трекінгу

  - **Особливості:** ES6+ polyfills, підтримка Promise, реалізація Symbol, протоколи ітераторів

### **Ключові Переваги Архітектури**

#### **1. Система Адаптерів на Базі Плагінів**
- **Розділення Проблем:** Кожний адаптер обробляє інтеграцію специфічного джерела попиту
- **Незалежна Розробка:** Адаптери можуть розроблятися та тестуватися ізольовано
- **Динамічне Завантаження:** Адаптери реєструються під час виконання дозволяючи гнучку конфігурацію
- **Стандартизований Інтерфейс:** Спільний API контракт забезпечує консистентну поведінку між адаптерами

#### **2. Модульна Стратегія Тестування**
- **Ізольовані Юніт Тести:** Кожний адаптер має виділений набір тестів з mock відповідями
- **Паралельне Виконання Тестів:** Mocha дозволяє одночасне запуск тестів для швидшого зворотного зв'язку
- **Тестування в Реальних Браузерах:** Karma запускає тести в справжніх браузерах для валідації сумісності
- **Інфраструктура Мокінгу:** Sinon надає комплексний мокінг для зовнішніх залежностей

#### **3. Оптимізація Часу Збірки**
- **Tree Shaking:** Webpack видаляє невикористаний код зменшуючи розмір bundle
- **Розділення Коду:** Динамічні імпорти дозволяють завантаження адаптерів на вимогу
- **Мініфікація:** Babel та Webpack мінімізують розмір production bundle
- **Source Maps:** Відлагодження розробки з маппінгом оригінальних рядків коду

#### **4. Управління Конфігурацією**
- **Глобальні Налаштування:** Централізована конфігурація через `config.js`
- **Адаптер-Специфічна Конфігурація:** Налаштування та перевизначення для кожного адаптера
- **Runtime Конфігурація:** Динамічні оновлення конфігурації через API
- **Environment-Aware:** Різні налаштування для середовищ розробки/production

  - Підтримка формату banner реклами з гнучким розміром
  - Real-time доставка контенту креативів

### **Основні Prebid Компоненти**

#### **Bid Manager**
- **Призначення:** Оркеструє процес аукціону та управляє життєвим циклом bid
- **Відповідальності:**
  - Валідує вхідні bid запити проти схем адаптерів
  - Управляє timeout аукціонів та грамотно обробляє збої адаптерів
  - Збирає та ранжує bid відповіді за CPM та іншими критеріями
  - Виконує переможний bid та обробляє рендеринг реклами
  - Відстежує аналітику аукціонів та метрики продуктивності

#### **Config Manager**
- **Призначення:** Обробляє глобальну та адаптер-специфічну конфігурацію
- **Особливості:**
  - Динамічні оновлення конфігурації без перезавантаження сторінки
  - Ізоляція та управління налаштуваннями специфічними для адаптерів
  - Контроль детальності цін для управління bid floor
  - Конфігурація timeout для оптимізації аукціону
  - Контролі режиму налагодження для розробки та усунення неполадок

#### **Utils Library**
- **Призначення:** Надає загальні утилітарні функції для розробки адаптерів
- **Ключові Функції:**
  - `parseSizesInput()` - Стандартизує обробку формату розмірів реклами
  - `deepAccess()` - Безпечний доступ до вкладених властивостей об'єктів
  - `flatten()` - Вирівнювання масивів для обробки відповідей
  - `isArray()` - Утиліти перевірки типів для валідації даних
  - `_map()` - Утиліти функціонального програмування для трансформації даних

## ⭐ Функції Системи

### **Real-Time Bidding (RTB)**
- **Паралельна Обробка Аукціонів:** Множинні адаптери торгуються одночасно для оптимальної прибутковості
- **Точність Часу в Мілісекундах:** Строге виконання timeout забезпечує продуктивність завантаження сторінки
- **Динамічне Ціноутворення Floor:** Настроювані мінімальні пороги bid для кожного рекламного блоку
- **Нормалізація Валют:** Підтримка мульти-валют з курсами конвертації в реальному часі
- **Оптимізація Прибутку:** Автоматичне ранжування bid та вибір переможця на основі eCPM

### **Підтримка Розвинених Форматів Реклами**

#### **Banner Реклама**
- **Гнучкі Розміри:** Підтримка стандартних IAB розмірів плюс кастомні розміри
- **Адаптивна Реклама:** Динамічні розміри на основі viewport та обмежень контейнера
- **Підтримка Rich Media:** HTML5, JavaScript та інтерактивні формати креативів
- **SafeFrame Інтеграція:** Безпечний iframe рендеринг для сторонніх креативів

## 🚀 Оптимізація Продуктивності

### **Мережева Продуктивність**
- **Connection Pooling:** Повторне використання HTTP з'єднань для зменшення overhead затримки
- **Request Batching:** Chunked bid запити мінімізують мережеві round trips (Adtelligent: 10 bids/запит)
- **Паралельна Обробка:** Одночасні запити адаптерів для максимальної пропускної здатності
- **Географічний CDN:** Балансування навантаження між регіональними endpoints для зменшення затримки

### **JavaScript Продуктивність**
- **Lazy Module Loading:** Адаптери завантажуються на вимогу зменшуючи початковий розмір bundle
- **Tree Shaking:** Webpack автоматично видаляє невикористані шляхи коду
- **Розділення Коду:** Динамічні імпорти для умовного завантаження адаптерів
- **Мініфікація:** Production збірки використовують розвинені техніки стиснення

## 🔒 Функції Безпеки

### **Захист від Cross-Site Scripting (XSS)**
- **Creative Sandboxing:** Всі сторонні креативи рендеряться в ізольованих iframes
- **Content Security Policy:** Строгі CSP заголовки запобігають неавторизованому виконанню скриптів
- **HTML Санітизація:** Серверна валідація та очищення HTML контенту креативів
- **Валідація Скриптів:** Whitelist-базоване схвалення для вбудованого JavaScript у креативах

### **Приватність Даних та Відповідність**

#### **GDPR Відповідність**
- **Управління Згодою:** Інтеграція з IAB Transparency & Consent Framework (TCF)
- **Обмеження Призначення:** Використання даних обмежене лише схваленими цілями
- **Мінімізація Даних:** Збір лише необхідних даних потрібних для показу реклами
- **Права Користувачів:** Підтримка запитів на доступ, виправлення та видалення
